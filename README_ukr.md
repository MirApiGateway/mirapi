# 🛡️ MirApi Proxy Gateway (Шлюз API-проксі)

[![Website](https://img.shields.io/badge/Website-mirapi.io-blue?style=flat-square)](https://mirapi.io)
[![License](https://img.shields.io/badge/License-MIT-green?style=flat-square)](LICENSE)
[![Status](https://img.shields.io/badge/Status-Active-brightgreen?style=flat-square)](https://github.com/MirApiGateway/mirapi/issues)

> **Високонадійний шлюз API-проксі для лінивих, але розумних розробників.**  
> Покращуйте свої інтеграції зі сторонніми API (Stripe, платіжні шлюзи, банківські API, провайдери ШІ) без жодних змін у коді. Просто спрямуйте свої запити на проксі-сервер та використовуйте спеціальні HTTP-заголовки, щоб миттєво додати стійкість корпоративного рівня.

---

## 🎁 Безкоштовний тариф та досвід розробника (Free Tier)
Щоб ви могли створювати, тестувати та запускати робочі навантаження без ризиків, MirApi пропонує **повністю безкоштовний тариф**:
* **10 000 безкоштовних запитів на місяць** (ліміт оновлюється щомісяця).
* **Кредитна картка не потрібна** для реєстрації.
* Працює на базі нашого високопродуктивного **вузла в Сінгапурі**, що забезпечує внутрішню затримку обробки **< 2 мс**.
* Зареєструйтеся за 10 секунд на [mirapi.io](https://mirapi.io) та почніть тестування за допомогою нашого публічного **Mock-сервера** за адресою `https://mock.mirapi.io`.

## 🎯 Практичні сценарії використання та рішення

### 1. Індія: Резервування UPI-платежів та каскадна маршрутизація (`X-Route-Key`)
* **Проблема:** Платежі UPI (Unified Payments Interface) в Індії часто зазнають затримок з боку банків або збоїв на боці платіжних шлюзів (наприклад, Razorpay, Paytm), через що мерчанти втрачають замовлення та клієнтів.
* **Рішення:** Визначте каскадний маршрут у базі даних. Якщо основний шлюз не відповідає протягом заданого ліміту (наприклад, `X-Attempt-Timeout: 2000ms`), MirApi silently та автоматично перемикається на резервний шлюз (наприклад, Cashfree або Paytm) за стратегіями пріоритету (`priority`) чи паралельної гонки (`race`). Жодних змін у логіці вашого коду.

### 2. Shopify: Доставка вебхуків та кастомний мапінг полів (`X-Webhook-Callback`)
* **Проблема:** Вебхуки Shopify втрачаються, якщо ваша CRM/ERP-система тимчасово не працює. Крім того, Shopify надсилає стандартний формат JSON, тоді як цільове API часто вимагає кастомної структури, що змушує писати окремий middleware.
* **Рішення:**
  * Спрямуйте вебхуки Shopify через MirApi за допомогою параметрів URL-запиту (оскільки Shopify не підтримує кастомні заголовки): `https://proxy.mirapi.io/v1/webhook?mirapi_key=YOUR_KEY&callback=https%3A%2F%2Fyour-erp.com%2Forders&retries=5`.
  * Трансформуйте вхідний JSON-запит під потрібну структуру на льоту на рівні проксі за допомогою правил `X-Extract-Map` (наприклад, мапінг `$.id=>order_id`) або каскадних маршрутів БД, щоб пристосувати його до цільового API.
  * Якщо кінцева ERP-система не працює, MirApi поміщає вебхук у чергу в Redis і надійно повторює спроби доставки протягом 24 годин.

## 🛠️ Швидкий старт та приклади інтеграції

Інтеграція MirApi **не вимагає встановлення жодних SDK**. Просто спрямуйте ваші запити через наш проксі-сервер (`https://proxy.mirapi.io/`) та додайте заголовки керування.

### 1. Звичайна маршрутизація з авто-повторами та джиттером
Надсилайте запити до цільового API та автоматично повторюйте спроби до 5 разів за алгоритмом **експоненціального бекоффу з джиттером** у разі помилок сервера (HTTP `5xx` або таймаутів).

```bash
curl -X POST https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: YOUR_MIRAPI_KEY" \
  -H "X-Target-URL: https://api.stripe.com/v1/charges" \
  -H "X-Retry-Count: 5" \
  -H "X-Retry-Delay: 300ms" \
  -H "Content-Type: application/json" \
  -d '{"amount": 1000, "currency": "usd"}'
```

### 2. Розумне кешування (Smart Cache при збоях)
Якщо стороннє API раптово виходить з ладу або повертає внутрішню помилку сервера `5xx`, MirApi перехоплює збій і віддає клієнту останню успішну відповідь із пам'яті.

```bash
curl -X GET https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: YOUR_MIRAPI_KEY" \
  -H "X-Target-URL: https://api.weatherapi.com/v1/current.json?q=Singapore" \
  -H "X-Smart-Cache: 300s"
```

### 3. Асинхронне виконання запитів (Черга вебхуків)
Миттєво звільняйте потік вашого додатку, отримавши статус `202 Accepted`. MirApi виконає запит у фоновому режимі через чергу воркерів і надішле результат на вказаний вами URL обробника вебхуків.

```bash
curl -X POST https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: YOUR_MIRAPI_KEY" \
  -H "X-Target-URL: https://api.openai.com/v1/chat/completions" \
  -H "X-Webhook-Callback: https://your-app.com/webhooks/openai" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4", "messages": [{"role": "user", "content": "Hello!"}]}'
```

### 4. Валідація PCI-DSS (Блокування карткових даних)
Запобігайте витоку сирих карткових даних. MirApi автоматично перевіряє тіло вхідних запитів за допомогою **алгоритму Луна**. Якщо виявлено сирі номери платіжних карток, запит негайно блокується на рівні проксі.

```bash
# Цей запит буде заблоковано:
curl -X POST https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: YOUR_MIRAPI_KEY" \
  -H "X-Target-URL: https://api.stripe.com/v1/charges" \
  -H "Content-Type: application/json" \
  -d '{"card_number": "4111111111111111"}'
```

* **Статус відповіді:** `400 Bad Request`
* **Тіло відповіді:** `"Error: Clear-text payment data detected. Please use tokenization (Stripe/Braintree tokens) instead of raw card numbers."`

### 5. Конфігурація через параметри URL (Альтернатива заголовкам)
Для клієнтів, які не дозволяють встановлювати власні HTTP-заголовки (наприклад, **вебхуки Shopify Admin**), будь-який заголовок конфігурації проксі можна передати безпосередньо через параметри запиту в URL.

> [!IMPORTANT]
> **Обов'язкове кодування URL (URL Encoding):** Будь-який URL, переданий всередині параметра запиту (наприклад, `target` або `callback`), має бути закодований (замініть `https://` на `https%3A%2F%2F` та `/` на `%2F`).

```bash
# Черга вебхуків та доставка суто за допомогою параметрів URL:
curl -X POST "https://proxy.mirapi.io/v1/webhook?mirapi_key=YOUR_API_KEY&callback=https%3A%2F%2Fyour-app.com%2Fwebhooks&retries=5&delay=200ms" \
  -H "Content-Type: application/json" \
  -d '{"event": "order_created"}'
```

Для отримання детальнішої інформації про мапінг параметрів запиту та приклади див. [Посібник з конфігурації параметрів URL](documentation/api-reference/api-reference;query-params.mdx).

## 🧪 Тестування за допомогою публічного Mock-сервера

Ми розгорнули публічний мок-сервер за адресою `https://mock.mirapi.io` спеціально для відлагодження та тестування. Ви можете проксіювати запити через MirApi, щоб миттєво симулювати різні успішні та помилкові відповіді.

### Доступні тестові ендпоінти:

| Шлях на Mock-сервері | Поведінка та сценарій тестування |
| :--- | :--- |
| `GET/POST https://mock.mirapi.io/post` | Дзеркально повертає вхідний JSON-запит (Тести POST, мапінг полів). |
| `GET https://mock.mirapi.io/delay/:seconds` | Навмисно затримує відповідь для перевірки таймаутів (напр. `X-Attempt-Timeout`). |
| `GET https://mock.mirapi.io/status/:code` | Повертає вказаний HTTP-статус (напр. `502`, `503` для тестів авто-повторів та failover). |
| `GET https://mock.mirapi.io/random` | Повертає випадковий `200 OK` або `502 Bad Gateway` (ідеально для тестування повторних спроб). |

### Приклад тесту: Цикл авто-повторів з Mock-сервером
Протестуйте поведінку авто-повторів безпечно. Проксі перехоплюватиме помилки `502` від мок-сервера і повторюватиме спроби до отримання успішного результату:

```bash
curl -X POST https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: YOUR_MIRAPI_KEY" \
  -H "X-Target-URL: https://mock.mirapi.io/random" \
  -H "X-Retry-Count: 4" \
  -H "X-Retry-Delay: 200ms"
```

## 🔒 Безпека та делегування секретів (Secrets Offloading)

Тримайте API-ключі сторонніх сервісів поза конфігураційними файлами та кодом вашого додатку:
* **Тимчасове зберігання (`X-Identity-Key`):** API-ключі зчитуються безпосередньо в оперативну пам'ять для виконання запиту і повністю видаляються без запису в логи.
* **Зашифроване зберігання (`X-Proxy-Master-Key`):** API-ключі зберігаються в базі даних у зашифрованому вигляді та розшифровуються проксі-сервером на льоту за допомогою майстер-ключа, переданого у цьому заголовку.
