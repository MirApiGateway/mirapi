## 🎯 Що таке MirApi?

**MirApi** — це надшвидкий API проксі-шлюз з мінімальною затримкою, написаний на Go. Він працює як прошарок між вашим додатком та сторонніми сервісами (такими як Stripe, OpenAI чи банківські системи). Змінивши лише базовий URL ваших API-запитів і додавши кастомні заголовки, ви отримуєте миттєвий доступ до розширених сценаріїв стійкості, безпеки, маршрутизації та кешування.

### ⚡ Вплив на продуктивність
* Власна затримка обробки запиту: **< 2мс**.
* Побудований на неблокуючій асинхронній архітектурі з використанням стандартної бібліотеки Go та швидкого кешування в Redis.

## 🚀 Основні фічі

### 1. 🛡️ Перевірка відповідності PCI-DSS (Hard Stop)
Захищає вашу інфраструктуру від складного аудиту та зайвої відповідальності за зберігання платіжних даних. Шлюз сканує вхідні JSON-запити у відкритому вигляді на наявність 13-16 значних номерів карт. При виявленні він застосовує перевірку за **алгоритмом Луна** і миттєво блокує запит.
* *Результат:* Номери карт у відкритому вигляді ніколи не потрапляють на ваші сервери та в логи.

### 2. 🔁 Автоматичні повтори запитів із джиттером
Запобігає збоям у роботі додатку через тимчасові проблеми з мережею. Автоматично повторює неуспішні запити (при HTTP-статусах `>= 500` або обривах зв'язку) за схемою **експоненціального бекоффу** та випадкового зміщення затримки (**джиттеру**), щоб не перевантажувати кінцевий сервер.

### 3. 🧠 Розумне кешування (Smart Cache)
Якщо сторонній сервіс раптом «приліг» або повертає помилку `5xx`, MirApi перехоплює цей збій і миттєво віддає останню успішну відповідь із пам'яті Redis.

### 4. 🔀 Каскадна маршрутизація та резервування (Failover)
Налаштовуйте основні та резервні точки доступу.
* **Стратегія пріоритету (Priority):** Запити виконуються послідовно. Якщо основний шлюз недоступний, проксі непомітно для клієнта перенаправить запит на резервного провайдера.
* **Стратегія перегонів (Race):** Запит надсилається на кілька адрес одночасно. Шлюз використовує найпершу успішну відповідь, а інші з'єднання скасовує.

### 5. ⚡ Асинхронні вебхуки (Режим черги)
Миттєво звільняє потоки вашого додатку, повертаючи статус `202 Accepted`. MirApi поміщає запит у високопродуктивну чергу воркерів на базі Redis, надійно виконує його (повторюючи спроби годинами за потреби) і надсилає результат на ваш URL вебхуку.

### 6. 🔒 Приховування секретів (Secret Offloading)
Зберігайте API-ключі сторонніх сервісів у безпеці. MirApi може розшифровувати платіжні ключі «на льоту» за допомогою майстер-пароля у заголовках або зчитувати їх із безпечних тимчасових циклів пам'яті.

## 💼 Де можна застосувати MirApi?

* **Стабільність платежів:** Запобігайте втраті клієнтів, коли Stripe, PayPal чи банківські сервіси мають тимчасові технічні труднощі.
* **Резервування LLM API:** Автоматично перемикайтеся на резервного AI-провайдера (наприклад, з Anthropic на OpenAI), якщо перевищено ліміти або сервіс недоступний.
* **Модернізація застарілих API:** Додайте сучасні функції (ідемпотентність, ліміти таймаутів, мапінг полів, кешування) до старих систем без переписування їхнього коду.
* **Захист від витоку даних:** Гарантуйте, що розробники чи тестові середовища випадково не відправлять номери карток у відкритому вигляді.

## 🛠️ Мінімальні приклади

Інтеграція MirApi **не потребує жодних SDK**. Просто змініть базовий URL на адресу вашого шлюзу MirApi та додайте потрібні заголовки.

### 1. Звичайне проксіювання з автоматичними повторами
Надіслати запит на httpbin та автоматично зробити до 3 повторних спроб у разі помилки.

```bash
curl -X POST https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: your_mirapi_client_key" \
  -H "X-Target-URL: https://httpbin.org/post" \
  -H "X-Retry-Count: 3" \
  -H "X-Retry-Delay: 200ms" \
  -H "Content-Type: application/json" \
  -d '{"status": "resilient"}'
```

### 2. Розумне рятівне кешування (Smart Cache)
Увімкнути кешування на 5 хвилин. Якщо Stripe тимчасово не працюватиме, MirApi поверне закешовану відповідь.

```bash
curl -X POST https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: your_mirapi_client_key" \
  -H "X-Target-URL: https://api.stripe.com/v1/charges" \
  -H "X-Smart-Cache: 5m" \
  -H "Content-Type: application/json" \
  -d '{"amount": 2000, "currency": "usd"}'
```

### 3. Асинхронний виклик через вебхуки
Передати виконання тривалого запиту у фон. MirApi миттєво поверне `202 Accepted`.

```bash
curl -X POST https://proxy.mirapi.io/ \
  -H "X-MirApi-Key: your_mirapi_client_key" \
  -H "X-Target-URL: https://httpbin.org/delay/10" \
  -H "X-Webhook-Callback: https://your-server.com/webhooks/mirapi" \
  -H "Content-Type: application/json" \
  -d '{"task": "background-process"}'
```
